## หลักฐานที่มี

### 1. CCTV จาก ATM สยาม (ชัดเจน)
- **ความยาว**: 2 นาที
- **คุณภาพ**: ชัดเจน เห็นหน้า
- **สิ่งที่เห็น**: สมชาย ถอนเงิน 500,000 บาท
- **การใช้งาน**: Search "@สมชาย" → เจอทันที

### 2. CCTV จาก Cafe มุมไม้ (48 ชั่วโมง)
- **ความยาว**: 48 ชั่วโมง
- **คุณภาพ**: กล้องมุมสูง ไม่เห็นหน้าชัด
- **สิ่งที่เห็น**: คนเดิน นั่ง แต่ไม่รู้ใครเป็นใคร
- **ความท้าทาย**: ต้องหาว่าใครเจอกันเมื่อไหร่

### 3. เอกสารทางการเงิน
- **สลิปโอนเงิน**: สมศรี → มิชชี่ (2 ล้าน)
- **สเตทเมนต์**: สมชาย ถอน 500K วันที่ 15 มี.ค.
- **สัญญา**: สมศรี ลงทุนกับ Icon Group

### 4. บันทึกโทรศัพท์
- **สมชาย → มิชชี่**: 14:35 น. ความยาว 2 นาที
- **มิชชี่ → สมศรี**: 14:25 น. ความยาว 5 นาที

## Workflow การสืบสวน

### Step 1: สร้าง Person Database

```
เจ้าหน้าที่:
1. เพิ่ม สมชาย → อัปโหลดรูปบัตรประชาชน + รูป ATM
2. เพิ่ก มิชชี่ → อัปโหลดรูปจากโซเชียล
3. เพิ่ม สมศรี → อัปโหลดรูปถ่าย
```

### Step 2: อัปโหลดและวิเคราะห์เอกสาร

```
เจ้าหน้าที่:
1. อัปโหลด สลิปโอนเงิน สมศรี
   ↓
   Azure DI อ่าน → เจอ "สมศรี โชคร้าย" → Link to person_id: p_003
   ↓
   เจอ "มิชชี่ รักดี" → Link to person_id: p_002
   ↓
   เจอจำนวนเงิน 2,000,000 บาท → Create transaction record

2. อัปโหลด สเตทเมนต์ สมชาย
   ↓
   Azure DI อ่าน → เจอการถอน 500,000 ที่ ATM สยาม 14:30
   ↓
   Link to person_id: p_001
```

### Step 3: อัปโหลด CCTV ATM (ชัดเจน)

```
เจ้าหน้าที่:
1. อัปโหลด CCTV ATM (2 นาที)
   ↓
   Twelve Labs Index
   ↓
   Search "@สมชาย" → เจอที่ timestamp 00:00:45
   ↓
   System auto-link: Video observation + Person p_001 + Bank record
```

### Step 4: อัปโหลด CCTV Cafe (48 ชม.)

```
เจ้าหน้าที่:
1. อัปโหลด CCTV 48 ชั่วโมง
   ↓
   Twelve Labs Index (ใช้เวลา)
   ↓
   Index เสร็จ

2. ค้นหา: "คนสามคนนั่งที่โต๊ะ"
   ↓
   Twelve Labs returns:
   - 00:04:23 (3 คนนั่งโต๊ะกลาง)
   - 00:12:15 (3 คนนั่งโต๊ะมุม)
   - 00:45:30 (3 คนนั่งโต๊ะหน้าต่าง)
   - 14:45:22 (3 คนนั่งโต๊ะมุม) ← ตรงเวลากับเหตุการณ์
   - ... (อีก 15 รายการ)

3. เจ้าหน้าที่ดู clip ที่ 14:45:22
   ↓
   เห็น: คนเสื้อแดง + คนผมยาว + คนอ้วน
   ↓
   เปรียบเทียบกับ Person database:
   - เสื้อแดง ≈ สมชาย (จากรูปถ่าย)
   - ผมยาว ≈ มิชชี่ (จากรูปถ่าย)
   - คนอ้วน ≈ สมศรี (จากรูปถ่าย)
   ↓
   เจ้าหน้าที่คลิก "ยืนยันบุคคล" → Link ทั้ง 3 คน
```

### Step 5: Cross-Reference

```
ระบบแสดง:

สมชาย @ 14:45:22 (Cafe)
├── 30 นาทีก่อน: ถอนเงิน 500K ที่ ATM (มี CCTV)
├── 10 นาทีก่อน: โทรหา มิชชี่ (call record)
└── อยู่ด้วยกัน: มิชชี่ + สมศรี

มิชชี่ @ 14:45:22 (Cafe)
├── 1 ชั่วโมงก่อน: รับโอน 2 ล้านจาก สมศรี (slip)
├── 10 นาทีก่อน: รับโทรศัพท์จาก สมชาย
└── อยู่ด้วยกัน: สมชาย + สมศรี

สมศรี @ 14:45:22 (Cafe)
├── 45 นาทีก่อน: โอนเงิน 2 ล้านให้ มิชชี่ (slip)
└── อยู่ด้วยกัน: สมชาย + มิชชี่
```

### Step 6: สรุป Timeline

```
ระบบสร้าง Timeline อัตโมัติ:

15 มี.ค. 2568
├── 14:00 - สมศรี โอนเงิน 2 ล้านให้ มิชชี่ [เอกสาร]
├── 14:30 - สมชาย ถอนเงิน 500K ที่ ATM [CCTV + เอกสาร]
├── 14:35 - สมชาย โทรหา มิชชี่ [call record]
└── 14:45 - ทั้ง 3 คน พบกันที่ Cafe [CCTV + link]
```

## สรุปสิ่งที่ระบบต้องมี

1. **Person Builder** - เก็บรูปหลายมุม + คำอธิบายลักษณะ
2. **Document Upload** - Azure DI อ่าน + ดึง entity
3. **Video Upload** - Twelve Labs index + search
4. **Scene Review UI** - ดู clip + ยืนยันบุคคล + เพิ่มโน้ต
5. **Cross-Reference** - เชื่อมโยงหลักฐานหลายแหล่ง
6. **Timeline View** - เรียงลำดับเหตุการณ์
