# แผนการพัฒนา DSI Intel Platform (MVP)

> เอกสารนี้บันทึกแผนการพัฒนาเวอร์ชัน MVP สำหรับ DSI Intel Platform
> เน้น User Workflow เป็นหลัก ไม่กังวลเรื่องเทคโนโลยีมาก

---

## สรุปโครงการ

แพลตฟอร์มช่วยสืบสวนคดีโดยเชื่อมโยงข้อมูลบุคคล เอกสาร และวิดีโอเข้าด้วยกัน

**หลักการสำคัญ:**
- เน้น MVP - ทำงานได้จริง ไม่ over-engineer
- ทุกอย่างแสดงเป็นภาษาไทย
- ไม่ตัดสินใจเรื่อง Tech Stack ตอนนี้
- สร้าง Mock Data เอง (DSI ไม่ให้ข้อมูลจริง)

---

## ฟีเจอร์หลัก (MVP)

### 1. ระบบจัดการบุคคล (Person Builder)

**สิ่งที่ทำได้:**
- เพิ่มบุคคลใหม่ พร้อมข้อมูลหลายภาษา (ไทย/อังกฤษ)
- อัปโหลดรูปภาพใบหน้าชัดเจน (front, profile, candid)
- บันทึกชื่อเล่น ชื่ออื่น หมายเลขโทรศัพท์ เลขบัตรประชาชน
- ระบุบทบาท: ผู้ต้องสงสัย, พยาน, ผู้เสียหาย

**ตัวอย่างข้อมูล:**
```
ชื่อ: สมชาย ใจดี / Somchai Jaidee
ชื่อเล่น: ชาย / Chai
บทบาท: ผู้ต้องสงสัย
รูปภาพ: [รูปหน้าตรง, รูปด้านข้าง]
```

---

### 2. ระบบอัปโหลดและวิเคราะห์เอกสาร (Azure DI)

**สิ่งที่ทำได้:**
- อัปโหลดเอกสาร PDF: สเตทเมนต์ธนาคาร, สัญญา, บัตรประชาชน, แชทล็อก
- ระบบ OCR อ่านข้อความอัตโนมัติ
- ดึง Entity สำคัญ: ชื่อบุคคล, เลขบัญชี, วันที่, จำนวนเงิน
- ลิงก์บุคคลในเอกสารกับฐานข้อมูลบุคคล

**เอกสารที่รองรับ:**
- สเตทเมนต์ธนาคาร (ไทย/อังกฤษ)
- สัญญาเงินกู้/การลงทุน
- บัตรประชาชน
- หลักฐานแชท (LINE, WhatsApp)

---

### 3. ระบบอัปโหลดและวิเคราะห์วิดีโอ (Twelve Labs)

**สิ่งที่ทำได้:**
- อัปโหลดวิดีโอ CCTV หรือคลิปจากโทรศัพท์
- Index วิดีโอเพื่อค้นหาได้
- ใช้ Entity API ค้นหาบุคคลในวิดีโอทั้งหมด
- ได้เวลาที่บุคคลปรากฏ (timestamp)

**หมายเหตุสำคัญ:**
- Twelve Labs รองรับเฉพาะวิดีโอ (ไม่รองรับไฟล์เสียงหรือ PDF โดยตรง)
- เสียงในวิดีโอจะถูกวิเคราะห์โดยอัตโนมัติเป็นส่วนหนึ่งของวิดีโอ

---

### 4. ระบบค้นหาข้ามข้อมูล (Cross-Search)

**สิ่งที่ทำได้:**
- พิมพ์ชื่อบุคคล → ค้นหาทั้งในเอกสารและวิดีโอ
- แสดงผลรวม: "สมชาย พบในเอกสาร 3 รายการ และวิดีโอ 5 ช่วงเวลา"
- คลิกดูรายละเอียดได้ทันที

**ตัวอย่างการค้นหา:**
```
ค้นหา: @สมชาย
ผลลัพธ์:
├── เอกสาร: สเตทเมนต์ธนาคาร SCB (15 มี.ค. 68)
├── เอกสาร: สัญญาเงินกู้ (10 มี.ค. 68)
├── วิดีโอ: CCTV ร้านกาแฟ @ 14:32:15
└── วิดีโอ: ATM สยามสแควร์ @ 16:45:00
```

---

### 5. ระบบธุรกรรมการเงิน (Financial Network)

**สิ่งที่ทำได้:**
- ดึงข้อมูลธุรกรรมจากสเตทเมนต์ธนาคาร
- สร้างเครือข่ายการโอนเงินระหว่างบุคคล
- แสดงเส้นทางการเงิน: A → B → C
- ค้นหาธุรกรรมที่เกี่ยวข้องกับบุคคล

**ตัวอย่างการใช้งาน:**
```
ค้นหา: เส้นทางเงินจาก @สมชาย
ผลลัพธ์:
สมชาย ──500,000 บาท──→ มิชชี่ ──300,000 บาท──→ สมหญิง
         (15 มี.ค. 68)       (16 มี.ค. 68)
```

## User Workflow หลัก

### Workflow 1: สร้างคดีและเพิ่มบุคคล

```
1. สร้างคดีใหม่
   ↓
2. เพิ่มบุคคลในคดี
   - กรอกชื่อ (ไทย/อังกฤษ)
   - อัปโหลดรูปใบหน้า
   - ระบุบทบาท
   ↓
3. บันทึก → ระบบสร้าง Person ID
```

### Workflow 2: อัปโหลดและวิเคราะห์เอกสาร

```
1. อัปโหลด PDF
   ↓
2. Azure DI อ่านเอกสารอัตโนมัติ
   ↓
3. ตรวจสอบ Entity ที่พบ
   - ชื่อบุคคล → ลิงก์กับ Person ที่มีอยู่
   - เลขบัญชี → บันทึกไว้
   - จำนวนเงิน → บันทึกธุรกรรม
   ↓
4. ยืนยันการลิงก์ → บันทึก Observation
```

### Workflow 3: อัปโหลดและวิเคราะห์วิดีโอ

```
1. อัปโหลดวิดีโอ
   ↓
2. Twelve Labs Index วิดีโอ
   ↓
3. ระบบค้นหาบุคคลในวิดีโอ
   - ใช้ Entity API กับรูปบุคคลที่มี
   - ได้เวลาที่พบบุคคล
   ↓
4. บันทึก Observation (person + video + timestamp)
```

### Workflow 4: ค้นหาข้อมูล

```
1. พิมพ์ชื่อบุคคลในช่องค้นหา
   ↓
2. ระบบค้นหาข้ามทุกแหล่งข้อมูล
   - เอกสาร (จาก Azure DI)
   - วิดีโอ (จาก Twelve Labs)
   - ธุรกรรมการเงิน
   ↓
3. แสดงผลรวมแบบกลุ่มตามประเภท
4. คลิกดูรายละเอียดแต่ละรายการ
```

### Workflow 5: ติดตามเส้นทางการเงิน

```
1. เลือกบุคคลต้นทาง
   ↓
2. ระบบแสดงธุรกรรมที่เกี่ยวข้อง
   ↓
3. คลิก "ติดตามเส้นทางเงิน"
   ↓
4. ระบบวิเคราะห์เครือข่ายการโอนเงิน
   ↓
5. แสดงกราฟเส้นทางเงิน
```

## Mock Data ที่ต้องสร้าง

เนื่องจาก DSI ไม่ให้ข้อมูลจริง เราต้องสร้างเอง:

### 1. Mock บุคคล (5-10 คน)
- ชื่อไทย/อังกฤษ
- รูปภาพ (ใช้ AI generate หรือ stock photo)
- ข้อมูลติดต่อ

### 2. Mock เอกสาร (10-20 รายการ)
- สเตทเมนต์ธนาคาร (template ธนาคารจริง)
- สัญญาเงินกู้
- บัตรประชาชน
- แชทล็อก

### 3. Mock วิดีโอ (5-10 คลิป)
- CCTV ร้านค้า
- กล้องหน้ารถ
- คลิปจากโทรศัพท์

**แนวคิดสร้าง Mock Data:**
- สร้างเรื่องราวคดีหลอก (fraud ring)
- กำหนดตัวละครและบทบาท
- สร้างเอกสาร/วิดีโอที่สอดคล้องกัน
- มีความสัมพันธ์ที่ตามหาได้

## สิ่งที่ยังไม่ทำใน MVP

- ❌ UI สวยงาม (ใช้แบบเรียบง่ายก่อน)
- ❌ Authentication ซับซ้อน
- ❌ Role-based access control
- ❌ Export report
- ❌ Real-time collaboration
- ❌ Mobile app
- ❌ AI วิเคราะห์อัตโนมัติทั้งหมด (ให้คนยืนยัน)

## คำถามที่ยังต้องคุยกัน

1. **คดีตัวอย่าง:** จะใช้เรื่องราวคดีอะไรเป็นโมเดล? (fraud, cybercrime, อื่น?)

2. **จำนวนข้อมูล:** ต้องการ Mock Data มากน้อยแค่ไหน?
   - บุคคล: 5, 10, 20 คน?
   - เอกสาร: 10, 50, 100 รายการ?
   - วิดีโอ: 5, 20, 100 ชั่วโมง?

3. **การค้นหา:** ต้องการ Filter อะไรบ้าง?
   - ช่วงวันที่?
   - ประเภทเอกสาร?
   - สถานที่?

4. **ธุรกรรมการเงิน:** ต้องการวิเคราะห์ลึกแค่ไหน?
   - แค่แสดงรายการ?
   - หรือต้องมีกราฟเครือข่าย?
