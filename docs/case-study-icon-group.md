# กรณีศึกษา: คดี Icon Group (Ponzi Scheme)

> เอกสารนี้ใช้เป็นต้นแบบสำหรับสร้าง Mock Data และออกแบบ User Workflow

---

## สรุปคดี

**ประเภท:** แชร์ลูกโซ่ / หลอกลงทุน (Ponzi Scheme)  
**บริษัท:** Icon Group (ไอคอน กรุ๊ป)  
**รูปแบบ:** ชักชวนลงทุนในตลาด Forex สัญญาให้ผลตอบแทนสูง  
**ผู้เสียหาย:** นับพันคน  
**มูลค่าความเสียหาย:** หลายพันล้านบาท

---

### 4. ธุรกรรมการเงิน (Financial Transactions)

#### โครงสร้างการเงิน
```
เงินจากผู้เสียหาย
       ↓
บัญชีนายหน้าระดับล่าง
       ↓
บัญชีนายหน้าระดับกลาง
       ↓
บัญชีบริษัทหลัก
       ↓
    [แยกเป็น]
       ├── จ่ายผลตอบแทนให้นายหน้า
       ├── จ่ายคืนนักลงทุนเก่า (Ponzi)
       └── ถอนไปบัญชีส่วนตัวผู้ก่อตั้ง
```

#### ข้อมูลธุรกรรมที่ต้องวิเคราะห์
- **จำนวนบัญชี:** 100-500+ บัญชี
- **ช่วงเวลา:** 1-3 ปี
- **มูลค่าธุรกรรม:** 10,000 - 50,000,000+ บาท
- **ประเภท:** โอนเงิน, ถอนเงินสด, โอนต่างประเทศ

## User Workflow สำหรับคดีนี้

### Workflow A: ติดตามเงิน (Follow the Money)

```
1. อัปโหลดสเตทเมนต์ธนาคารบริษัท
   ↓
2. Azure DI อ่านและดึงรายการธุรกรรม
   ↓
3. สร้างบุคคลจากชื่อในบัญชี
   ↓
4. วิเคราะห์เส้นทางเงิน
   - ใครโอนเงินเข้าบริษัท?
   - บริษัทโอนเงินไปให้ใคร?
   - มีลูกโซ่กี่ชั้น?
   ↓
5. แสดงกราฟเครือข่ายการเงิน
```

### Workflow B: หาพยานหลักฐาน (Find Evidence)

```
1. อัปโหลดวิดีโอสัมมนาทั้งหมด
   ↓
2. Twelve Labs Index วิดีโอ
   ↓
3. เพิ่มรูปผู้ต้องสงสัยใน Person Builder
   ↓
4. ค้นหา: "@ซีอีโอ พูดคำว่า 'ผลตอบแทน'"
   ↓
5. ได้เวลาที่พูด → คลิปหลักฐาน
```

### Workflow C: เชื่อมโยงบุคคล (Link Persons)

```
1. จากแชท LINE ที่ยึดได้
   ↓
2. Azure DI อ่านแชท
   ↓
3. พบชื่อ "พี่ชาย" ถูกกล่าวถึงบ่อย
   ↓
4. ค้นหา @พี่ชาย ในวิดีโอ CCTV
   ↓
5. พบว่าเป็นผู้ถือหุ้นรายใหญ่
   ↓
6. สร้างโครงสร้างองค์กร
```

### Workflow D: สร้าง Timeline

```
1. รวมข้อมูลจากทุกแหล่ง
   - วันที่ในสัญญา
   - วันที่ธุรกรรม
   - วันที่ในวิดีโอ
   ↓
2. เรียงลำดับเหตุการณ์
   ↓
3. แสดง Timeline คดี
   - เมื่อไหร่เริ่มรับเงิน?
   - เมื่อไหร่หยุดจ่ายผลตอบแทน?
   - ใครถอนเงินก่อนวิกฤต?
```
